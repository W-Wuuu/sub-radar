# Agent B — 监控&优化层 设计文档

## 角色
质量保障者。监控Agent A的产出、分析数据、驱动迭代、异常告警。

## 核心职责

| 职责 | 实施方式 | 频率 |
|------|---------|------|
| 站点健康检查 | HTTP状态码 + 页面加载 | 每日 |
| 内容质量审查 | 死链检查、格式验证 | 每次部署后 |
| 流量数据分析 | Cloudflare Analytics | 每周 |
| 实验数据追踪 | 虚拟MVP指标更新 | 每周 |
| 用户反馈分析 | 实验问卷 + 预注册留言 | 每周 |

## 监控仪表盘

### 技术指标
| 指标 | 健康阈值 | 告警阈值 |
|------|---------|---------|
| 站点可用性 | 100% | < 99% |
| 页面加载时间 | < 2s | > 5s |
| 死链数量 | 0 | > 0 |
| 部署成功率 | 100% | < 100% |

### 业务指标（虚拟MVP阶段）
| 指标 | 健康阈值 | 告警阈值 |
|------|---------|---------|
| 周PV环比 | ≥ 0% | 连续2周下降 |
| 实验完成率 | ≥ 30% | < 15% |
| "愿意授权"比例 | ≥ 50% | < 30% |

### 正式运营后追加（见 agent-b-metrics memory）
| 指标 | 健康阈值 | 预警阈值 |
|------|---------|---------|
| WAU | 环比 ≥ 0% | 连续2周下降 |
| NPS | ≥ 30 | < 10 |
| 月流失率 | < 5% | > 8% |

## 告警响应

```
指标异常
    │
    ▼
自动分析（AI诊断可能原因）
    │
    ▼
严重度判定
    │
    ├── 低：记录到周报
    ├── 中：自动修复尝试 + 周报标注
    └── 高：立即通知CEO-Agent → 评估是否上报架构师
```

## 与Agent A的协作

Agent B 通过以下方式向 Agent A 提供反馈：

```
Agent B（监控）
    │
    │ 周报 / 告警 / 优化建议
    ▼
CEO-Agent（统筹）
    │
    │ 调整内容策略 / 修复问题 / 优化方向
    ▼
Agent A（执行）
```
